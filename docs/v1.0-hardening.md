# v1.0 hardening and release matrix

This document tracks the remaining work to move `ovnflow` from the broad v0.2
API surface to a v1.0 release candidate.

## Acceptance gates

| Area | v1.0 requirement | Current status |
| --- | --- | --- |
| Module path | Module path, repository URL, examples, and release tags resolve through `github.com/firstmeet/ovnflow` | In progress |
| Public API | Keep v0.1/v0.2 source compatibility while documenting stable entrypoints | In progress |
| OVN Northbound | Fluent lifecycle for major NB tables, UUID-safe delete, map/set mutation by default | In progress |
| OVN Southbound | Typed list/get/watch for main runtime tables plus guarded generic writes | In progress |
| Open_vSwitch | Fluent lifecycle for Bridge, Port, Interface, Controller, Manager, Mirror, QoS, Queue, Flow_Table, NetFlow, sFlow, IPFIX, SSL, AutoAttach | In progress |
| Runtime schema | Required columns fail with `ErrorInvalidSchema`; optional columns degrade without panic | In progress |
| Concurrency | Watch fanout/cancel/overflow must not panic, block forever, or leak long-lived goroutines | In progress |
| Transactions | All production writes use `libovsdb` transactions and preserve external map/set keys | In progress |
| Deletes | Delete paths select UUIDs first, clean UUID references, and only require target deletes to affect rows | In progress |
| Tests | `go test ./...`, `go vet ./...`, examples, integration, and CI must pass | In progress |
| Release | Green CI on release branch, `v1.0.0` tag, GitHub release notes, pkg.go.dev discoverability | Pending |

## Known gaps

- Define and test the exact watch ordering contract between initial snapshots
  and live cache events. Current watches support fanout, cancel, overflow, and
  race-clean manager initialization, but the initial/live ordering semantics
  still need a v1.0 API decision.
- Harden NB generic reference cleanup for map-valued UUID references. Current
  delete paths select target UUIDs first and clean standard UUID-set references;
  map-valued reference cleanup needs the same precision as OVS map cleanup.
- Broaden unit tests for generic `TableRef` create/ensure/update/delete/list/watch
  across NB, SB, and OVS.
- Add operation-shape tests for SB typed list/get/watch wrappers and OVS extended
  fluent table helpers.
- Decide whether versioned file names such as `nb_v02*.go`, `nb_v2.go`,
  `sb_v2.go`, and `ovs_v2.go` should be renamed before v1.0 to domain names.
- Add a release workflow that validates a pushed `v*` tag before creating a
  GitHub release.
- Run race tests on a CGO-capable Linux runner; Windows local Go may not have
  CGO enabled.

## Release checklist

1. Confirm the working tree is clean on the release branch.
2. Run local validation:
   - `go test -count=1 ./...`
   - `go vet ./...`
   - `go test -run Example ./...`
   - `go test -bench=. -benchmem ./...`
3. Run Linux race validation:
   - `go test -race -count=1 ./...`
4. Run integration validation with reachable OVN/OVS endpoints:
   - `go test -tags=integration -count=1 -v ./...`
5. Push the branch and require GitHub Actions success.
6. Create and push `v1.0.0`.
7. Confirm `go list -m github.com/firstmeet/ovnflow@v1.0.0` and the pkg.go.dev
   module page resolve.
