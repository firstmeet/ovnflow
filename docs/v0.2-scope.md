# v0.2 scope and acceptance matrix

`ovnflow` v0.2 expands the v0.1 proof surface into a broader fluent SDK for
OVN Northbound, OVN Southbound, and local Open vSwitch OVSDB workflows. The
release keeps the v0.1 contract intact: production paths use `libovsdb`,
ordinary tests are local, and integration tests are enabled only by build tags
and environment variables.

## API coverage

| Area | Tables | Operations |
| --- | --- | --- |
| Shared fluent runtime | runtime-selected tables | `Create`, `Ensure`, `Update`, `Delete`, `Get`, `List`, `Watch`, `WithColumn`, optional columns, map/set mutations |
| Schema runtime | all connected schemas | table/column capability checks, required condition validation, reference-column discovery, typed `ErrorInvalidSchema` |
| OVN Northbound L2 | `Logical_Switch`, `Logical_Switch_Port` | v0.1-compatible lifecycle, subnet/external IDs, port add/delete, get/list/watch through shared table API |
| OVN Northbound L3 and policy | `Logical_Router`, `Logical_Router_Port`, `ACL`, `NAT`, `QoS` | handwritten fluent lifecycle builders, named/compound identities, map/set mutation helpers |
| OVN Northbound services | `Load_Balancer`, `DHCP_Options`, `DNS`, `Meter`, `Meter_Band` | handwritten lifecycle builders for VIPs, options, records, metering, and external IDs |
| OVN Northbound groups and HA | `Port_Group`, `Address_Set`, `Gateway_Chassis`, `HA_Chassis`, `HA_Chassis_Group`, `BFD` | handwritten lifecycle builders and UUID-reference helpers |
| OVN Southbound observability | `Chassis`, `Port_Binding`, `Datapath_Binding`, `Logical_Flow`, `MAC_Binding`, `FDB`, `Multicast_Group`, `Service_Monitor`, `RBAC_Role`, `RBAC_Permission`, `Meter`, `Meter_Band`, `DNS`, `BFD` | typed list/get/watch APIs plus generic table read/watch fallback |
| Open_vSwitch local setup | `Open_vSwitch`, `Bridge`, `Port`, `Interface`, `Controller`, `Manager`, `Mirror`, `QoS`, `Queue`, `Flow_Table`, `NetFlow`, `sFlow`, `IPFIX`, `SSL`, `AutoAttach` | v0.1-compatible bridge/port lifecycle, extended table fluent APIs, typed bridge/port/interface reads and watches |

Version-specific or rare schema columns can be reached through `Table`,
`TableBy`, `WithColumn`, `WithOptionalColumn`, `MutateMap`, and `MutateSet`
without adding generated code.

## Safety and behavior

| Requirement | Evidence |
| --- | --- |
| Production OVSDB access goes through `libovsdb` | production code imports `libovsdb`; `internal/ovsdbjson` is used by integration tests only |
| Runtime schema differences are handled explicitly | builders validate required tables, required columns, and condition columns before execution |
| Optional columns degrade gracefully | `WithOptionalColumn` and existing-column filtering skip missing optional columns |
| Map/set updates avoid overwriting other controllers | ensure/update paths separate row updates from OVSDB `mutate` operations |
| Deletes do not rely on names after selection | v0.2 delete paths select `_uuid`, unreference UUIDs where possible, then delete by `_uuid` |
| Watch supports several subscribers | shared watch manager fan-outs row events, supports cancel, and reports overflow as `ErrorPartial` |
| v0.1 API remains compatible | existing logical switch and bridge builders are preserved and covered by compatibility tests |

## Test matrix

| Check | Command or evidence |
| --- | --- |
| Windows/local unit tests without WSL or Docker | `go test -count=1 ./...` |
| Static analysis | `go vet ./...` |
| Example compilation | `go test -run Example ./...` |
| Benchmark smoke test | `go test -bench=. -benchmem ./...` |
| Integration tag without endpoints | `go test -tags=integration -count=1 -v ./...` skips endpoint tests |
| Windows to WSL endpoints | set `OVNFLOW_OVS_ADDR`, `OVNFLOW_OVN_NB_ADDR`, `OVNFLOW_OVN_SB_ADDR` and run `go test -tags=integration -count=1 -v ./...` |
| Read-only v0.2 schema readiness | add `OVNFLOW_V02_SCHEMA_CHECKS=1` to validate NB, SB, and OVS table/column coverage |
| Docker/Linux CI | GitHub Actions unit jobs plus Docker Compose-backed integration job |

## Integration environment

The base integration variables are shared by Windows + WSL and Linux/Docker:

```powershell
$env:OVNFLOW_OVS_ADDR="tcp:172.27.192.120:6640"
$env:OVNFLOW_OVN_NB_ADDR="tcp:172.27.192.120:6641"
$env:OVNFLOW_OVN_SB_ADDR="tcp:172.27.192.120:6642"
go test -tags=integration ./...
```

Read-only v0.2 schema checks add one opt-in variable:

```powershell
$env:OVNFLOW_V02_SCHEMA_CHECKS="1"
go test -tags=integration ./...
```

Mutation integration tests use prefixed resource names, caller-owned markers,
and dedicated bridges. They remain guarded by endpoint reachability and safety
configuration:

```powershell
$env:OVNFLOW_TEST_PREFIX="ovnflow-it-"
$env:OVNFLOW_TEST_BRIDGE="br-ovnflow-it"
go test -tags=integration ./...
```

If endpoints are not reachable, integration tests skip and print the WSL checks:
`ss -lntp | grep -E '6640|6641|6642'`,
`ovs-vsctl set-manager ptcp:6640:0.0.0.0`,
`ovn-nbctl set-connection ptcp:6641:0.0.0.0`, and
`ovn-sbctl set-connection ptcp:6642:0.0.0.0`.

## Release notes

- Push the release branch as `v0.2`.
- Keep `RawNB`, `RawSB`, and `RawOVS` available for unsupported edge cases.
- Do not stage `.gocache/` or other local build artifacts.
- Run the local matrix before pushing; run WSL or Docker integration when
  endpoints are reachable.
