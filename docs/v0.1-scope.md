# v0.1 scope and acceptance matrix

`ovnflow` v0.1 is intentionally small: it proves the fluent API shape, typed
errors, libovsdb-only production paths, and Windows + WSL integration workflow
before adding every OVN table.

## API coverage

| Area | Tables | Supported operations |
| --- | --- | --- |
| OVN Northbound | `Logical_Switch`, `Logical_Switch_Port` | `Create`, `Ensure`, `Delete`, `GetLogicalSwitch`, subnet/addresses/external IDs |
| OVN Southbound | `Chassis`, `Port_Binding`, `Datapath_Binding` | typed list APIs, port-binding watch, chassis watch |
| Open_vSwitch | `Open_vSwitch`, `Bridge`, `Port`, `Interface` | ensure bridge, add port/interface, delete port/interface, delete bridge tree |

Out of scope for v0.1: ACL, NAT, router, load balancer, DHCP, QoS, advanced
port types, and full OVN schema generation. These should be added as explicit
packages or builders after the initial contract is stable.

## Required acceptance checks

| Requirement | Evidence |
| --- | --- |
| Normal Windows tests do not need WSL or Docker | `go test ./...` |
| Integration endpoints are environment-driven TCP addresses | `OVNFLOW_OVS_ADDR`, `OVNFLOW_OVN_NB_ADDR`, `OVNFLOW_OVN_SB_ADDR`; non-`tcp:` integration endpoints are rejected |
| Unreachable WSL endpoints skip integration tests | integration helpers skip with `ss -lntp` and `set-manager/set-connection` guidance |
| OVN NB lifecycle works | integration creates/ensures/gets/deletes LS and LSP and checks UUID references |
| Local OVS lifecycle works | integration creates/ensures/deletes Bridge, Port, Interface and checks UUID references |
| OVS custom interface cleanup works | integration deletes a port whose interface has a different name and checks no orphan rows remain |
| OVN SB read/watch works | integration lists chassis, port bindings, datapaths, starts watch, then cancels cleanly |
| Docker is WSL/Linux CI only | `compose.yaml` and Linux CI job expose ports 6640/6641/6642; Windows job runs unit tests only |
| Publishable module | `go.mod`, `LICENSE`, package docs, README examples, and tagged release workflow |
